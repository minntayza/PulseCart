# Sketch 001: Product Feed

## Design Question
How should product browsing and the personalized agent-powered feed feel? This is the customer-facing view — the "user action" side of the demo split-screen.

## How to View
open .planning/sketches/001-product-feed/index.html

## Variants
- **A: Classic Grid** — Amazon-style responsive product grid with filter chips. Familiar e-commerce pattern. Agent badges are subtle additions.
- **B: Agent-Powered Feed** — Social media style feed with agent insight banner at top. Recommendations feel curated, not just listed.
- **C: Split Browse** — Left sidebar with categories + agent recommendations, right panel with product grid. More structured, dashboard-like.

## What to Look For
- Which layout feels most natural for a hackathon demo (quick to understand)?
- Do the agent badges (Agent Pick, Trending, Great Match) feel integrated or bolted on?
- Does Variant B's feed feel too slow for a 4-minute demo, or does the curated feel add value?
- Does Variant C's sidebar make the agent recommendations more visible?
- Search filtering works in all variants — try typing "gaming" or "chair"
